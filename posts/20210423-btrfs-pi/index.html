Raspberry Pi 4 - Ubuntu 20.04 w/Btrfs root

Apr 23, 2021 21:18 · 292 words · 2 minute read pi btrfs

How to convert a root filesystem on a Raspberry Pi 4 b running Ubuntu 20.04 64 bit to Btrfs in a few simple steps.

Why

First, why should I care about doing this.

  • Btrfs is SSD-aware, even on a SD card. The copy on write filesystem will ensure that your writes will be spread out and decrease wear leveling.
  • You can enable a cpu friendly, zstd, compression that decreases the amount of data written to your SD card.

How

  1. Flash Ubuntu on the SD card as usual.
  2. Boot once as usual and let the filsystem expand to the space of the SD card.
  3. Log into the Raspberry Pi.
  4. Tell boot to read from a Btrfs filesystem on next boot 
    sudo sed -i 's/ext4/btrfs/' /boot/firmware/cmdline.txt
  5. Update /etc/fstab with new fileystem, and ensure minimum writes to SD card. 
    sudo sed -E -i 's|(.*)ext4.*|\1btrfs compress=zstd:1,noatime,nodiratime 0 0|' /etc/fstab
  6. Shutdown your system. 
    sudo shutdown -h now
  7. Take out your SD card and put it into your Linux workstation or laptop. Convert the filesystem on the SD card. If the SD cards ext4 partition are /dev/sda2 then the command will be as follows. Let the command finish. 
    sudo btrfs-convert -p /dev/sda2
  8. Put the SD card back in the Raspberry Pi and boot. Voila! You have a Btrfs root filesystem.
  9. Log into the Rasberry Pi for some cleanup.
  10. Remove the old ext2 backup. 
    sudo btrfs sub del /ext2_saved
  11. Defrag and compress already existing OS. 
    sudo btrfs fi def -czstd -r -v /

Congratulation! Now you have a Btrfs root filesystem with compression. Writes will take less space, and wear leveling is down. Note that you also have saved space by running a df -h /. Enjoy an SD card that will last longer than you are used to.
